# samples/ 測試檔案

這些是**空檔案**，用於：

1. **單元測試** - 測試 `extract_number()` 番號解析邏輯
2. **新手教學** - 拖放進應用程式測試搜尋功能

---

## 資料夾說明

| 資料夾 | 用途 |
|--------|------|
| `basic/` | 新手測試用，標準格式，可搜尋到結果 |
| `real_world/` | 真實世界常見格式 |
| `suffix/` | 後綴處理測試（-C, -UC, -CD1） |
| `special_format/` | 特殊片商格式 |
| `tricky/` | 刁鑽邊界情況 |
| `noise/` | 雜訊干擾（網站浮水印等） |
| `edge_case/` | 極端邊界情況 |
| `invalid/` | 應返回 None 的無效檔名 |

---

## 已知問題

### ❌ 假番號（永遠找不到）

這些是測試用的假番號，不存在於真實世界：

| 檔名 | 說明 |
|------|------|
| `ABC-123.mkv` | 假番號，用於測試格式解析 |
| `FC2-PPV-123456.avi` | 假 FC2 番號 |
| `random_movie_2024.mp4` | invalid/ 測試用 |
| `123456.mkv` | invalid/ 測試用 |
| `movie.avi` | invalid/ 測試用 |

### ⚠️ 真實番號但目前不支援（待優化）

這些番號真實存在，但 `extract_number()` 目前的 regex 不支援：

| 檔名 | 問題 | 解決方案 |
|------|------|----------|
| `T28-103.avi` | 前綴含數字，不符合 `[A-Za-z]{2,6}` | 修改 regex 支援數字前綴 |
| `1PONDO-010120_001.mp4` | 一本道特殊格式 | 新增專用 pattern |
| `CARIBPR-010120-001.mkv` | 加勒比特殊格式 | 新增專用 pattern |

**追蹤 Issue**: 未來新增來源或優化搜尋規則時處理

---

## 預期結果

`expected_results.json` 定義了每個檔名的預期解析結果：
- 有值 = 預期提取出的番號
- `null` = 預期返回 None

---

## 給新手

想測試搜尋功能？請使用 `basic/` 資料夾內的檔案，這些番號可以搜尋到真實結果。
